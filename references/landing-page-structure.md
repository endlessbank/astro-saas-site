# Landing Page Structure

Homepage sections in order. Each section = one Astro component.

## 1. Header
- Logo (SaaS name)
- Nav: Home, Blog, FAQ, Pricing (anchor)
- CTA button (top right)

## 2. Hero Section
- **Headline**: Outcome-focused, short. Stop the scroll.
- **Subheading**: How it works / who it's for / what to expect
- **Hero image**: Placeholder (screenshot area)
- **CTA button**: Action-oriented ("Start Free", "Try Demo")

## 3. Social Proof (Early)
- Logos or "Used by X+ users"
- Short testimonial or tweet screenshot placeholder

## 4. Benefits (3 blocks)
Each benefit block:
- Icon placeholder
- Benefit headline (outcome, not feature)
- 1-2 line description

## 5. Social Proof (Second)
- Another testimonial
- Different format (quote with avatar)

## 6. Process (How It Works)
3 steps:
- Step number
- Title
- Short description
- Optional icon

## 7. Pricing
- Monthly/Yearly toggle (vanilla JS)
- 2 plan cards side by side
- Each card: name, price, feature list, CTA button
- Highlighted plan has accent border/background

## 8. CTA (After Pricing)
- Headline reinforcing value
- Single button

## 9. About Us
- Who we are (1-2 sentences)
- Why we built this (1-2 sentences)
- Optional founder image placeholder

## 10. Social Proof (Deep)
- Longer testimonial
- Customer name + role + company

## 11. FAQ Preview
- 3-4 top questions (accordion or list)
- Link to full FAQ page

## 12. Final CTA
- Last chance headline
- Button
- No new information

## 13. Footer
- Logo + tagline
- Links: Home, Blog, FAQ, Privacy, Terms
- Copyright
- Social icons (optional)

## Placeholder Content Guidelines

Use these patterns for placeholder text:
- Headlines: "[SAAS_NAME] helps you [outcome]"
- Subheadings: "The easiest way to [action] without [pain point]"
- Benefits: Focus on time saved, money saved, stress avoided
- Testimonials: "Since using [SAAS_NAME], we've [specific result]..."
- Process steps: "Sign up", "Connect your...", "Watch it work"
